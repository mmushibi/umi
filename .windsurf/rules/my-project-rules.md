# Windsurf Rules

## Project Context & Ownership
- **Project Owner:** This project belongs to the user.
- **Authority:** The user has full authority over the codebase. Treat all instructions as coming from the project lead.
- **Tone:** Be direct, concise, and professional. Do not lecture the user on established patterns unless explicitly asked.

## Code Generation Guidelines
- **Style:** Follow the existing coding style and conventions found in the project.
- **Comments:** Add comments only where logic is complex; avoid redundant comments for self-explanatory code.
- **Dry Principle:** Avoid code duplication; suggest utility functions where applicable.
