## Introduction

The UmiHealth Operations Portal is designed for efficient management of pharmacy inventory, procurement, logistics, and supplier relationships. This comprehensive system enables operations staff to maintain optimal stock levels, manage supply chains, and ensure product quality and availability.

### Key Features
- Real-time inventory management
- Automated procurement workflows
- Stock transfer management
- Supplier relationship management
- Quality control processes
- Analytics and reporting
- Compliance tracking

## Getting Started

### System Requirements

### Inventory Adjustments
#### Stock Updates
1. Navigate to **Inventory → Stock Updates**
2. Select adjustment type:
   - **Stock In**: New inventory received
   - **Stock Out**: Inventory removed
   - **Transfer**: Stock moved between locations
   - **Adjustment**: Correction or write-off
   - **Return**: Product returned to supplier
3. Enter product details:
   - Product name/code
   - Batch number
   - Expiry date
   - Quantity
   - Reason for adjustment
4. Verify information
5. Submit and document

#### Physical Stock Counts
1. Navigate to **Inventory → Stock Count**
2. Select counting area/category
3. Generate count sheet
4. Perform physical count
5. Enter counted quantities
6. Compare with system records
7. Investigate discrepancies
8. Post adjustments
9. Generate variance report

### Location Management

- Controlled substance cabinets
- Quarantine areas

#### Location Transfers
1. Navigate to **Inventory → Location Transfer**
2. Select source and destination locations
3. Choose products to transfer
4. Enter quantities
5. Verify transfer details
6. Execute transfer
7. Update location records

## Troubleshooting

### Common Issues

#### Inventory Management
- **Issue**: Stock count discrepancies
- **Solution**:
  1. Verify counting procedures
  2. Check recent transactions
  3. Review system logs
  4. Investigate potential theft
  5. Document findings

- **Issue**: System shows incorrect stock levels
- **Solution**:
  1. Refresh inventory data
  2. Check for pending transactions
  3. Verify batch records
  4. Conduct physical count
  5. Update system records

#### Procurement Issues
- **Issue**: Order not received on time
- **Solution**:
  1. Contact supplier immediately
  2. Verify shipment status
  3. Check tracking information
  4. Arrange alternative supply
  5. Document delay

- **Issue**: Received incorrect products
- **Solution**:
  1. Document discrepancies
  2. Contact supplier
  3. Arrange return/replacement
  4. Update order records
  5. Process corrections

#### System Issues
- **Issue**: System running slowly
- **Solution**:
  1. Check internet connection
  2. Clear browser cache
  3. Close unnecessary applications
  4. Restart browser
  5. Contact IT support

### Error Messages
- **Insufficient Stock**: Product quantity not available
- **Duplicate Order**: Order already exists
- **Invalid Supplier**: Supplier not approved
- **Expiry Date Warning**: Product approaching expiry
- **Transfer Failed**: Stock transfer cannot be completed
