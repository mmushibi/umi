## COMMON QUESTIONS

Q: Why do I need to add secrets manually?
A: Security - we don't want secrets stored in the repository. GitHub Secrets
   keeps them encrypted and only accessible during CI/CD.

Q: Can I use the same secret for all environments?
A: NO! Create different secrets for Development, Staging, and Production.

Q: What if I lose my secrets?
A: Regenerate them and update GitHub Secrets. The old values will be invalid.

Q: How often should I rotate secrets?
A: Every 90 days for maximum security. More frequently in high-risk scenarios.

Q: What if CI/CD fails?
A: Check the GitHub Actions logs. Most common issues are missing secrets or
   incorrect format. See PRODUCTION_DEPLOYMENT_GUIDE.md â†’ Troubleshooting
